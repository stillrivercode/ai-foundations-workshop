# Common Prompting Mistakes

<div class="mt-4">

### Don't Do This

<div class="space-y-4 text-sm">

<div class="p-2 bg-red-500/10 rounded">
<carbon-close class="text-red-400" /> **Vague requests**
<div class="opacity-75">"Make it better"</div>
</div>

<div class="p-2 bg-red-500/10 rounded">
<carbon-close class="text-red-400" /> **No context**
<div class="opacity-75">"Fix the bug" (what bug?)</div>
</div>

<div class="p-2 bg-red-500/10 rounded">
<carbon-close class="text-red-400" /> **Too much at once**
<div class="opacity-75">"Build entire app"</div>
</div>

<div class="p-2 bg-red-500/10 rounded">
<carbon-close class="text-red-400" /> **Assuming knowledge**
<div class="opacity-75">"Use our standard pattern"</div>
</div>

</div>

</div>

::right::

<div class="pl-8 mt-4">

### Do This Instead

<div class="space-y-4 text-sm">

<div class="p-2 bg-green-500/10 rounded">
<carbon-checkmark class="text-green-400" /> **Specific criteria**
<div class="opacity-75">"Reduce complexity, add error handling"</div>
</div>

<div class="p-2 bg-green-500/10 rounded">
<carbon-checkmark class="text-green-400" /> **Rich context**
<div class="opacity-75">"In this Express route, line 23..."</div>
</div>

<div class="p-2 bg-green-500/10 rounded">
<carbon-checkmark class="text-green-400" /> **Incremental tasks**
<div class="opacity-75">"First, design the data model"</div>
</div>

<div class="p-2 bg-green-500/10 rounded">
<carbon-checkmark class="text-green-400" /> **Explicit patterns**
<div class="opacity-75">"Use repository pattern like [example]"</div>
</div>

</div>

</div>

<div class="absolute bottom-12 left-12 right-12">
<div class="text-sm p-3 bg-blue-500/10 rounded-lg">
<carbon-idea class="inline text-blue-400" /> Rule of thumb: If a junior developer couldn't follow your prompt, neither can the AI.
</div>
</div>
